把你手上**已有的、可信的**資料丟進這個目錄，agent 在開始研究前會優先讀取，並把這些檔案視為**最高優先級的資料源**（高於 web 搜尋結果）。

這是把 AI 從「不可靠的網路抓取者」轉成「可靠的資料整合者」的關鍵設計。AI 不擅長突破付費牆、繞過 Cloudflare、解析動態網站；但它非常擅長把你給它的資料消化、交叉比對、結構化輸出。

## 怎麼用

### 1. 建立目標子目錄

每個分析目標一個目錄。中文原樣，不需要羅馬化：

```
input/
├── 個人理財 SaaS A/
├── 上市遊戲營運商 C/
└── 半導體設備/
```

子目錄名稱**建議與你輸入給 `/recon`、`/company`、`/industry` 的目標名稱完全一致**，agent 才能自動掃描到。

### 2. 把檔案丟進去

支援的檔案類型：

| 類型 | 副檔名 | 用途範例 |
|------|--------|---------|
| PDF | `.pdf` | 年報、ESG 報告書、簡報、研究報告、白皮書 |
| 圖片 | `.png` `.jpg` | 商工登記網站截圖、LinkedIn 截圖、媒體報導截圖、財務儀表板截圖 |
| 文字 | `.md` `.txt` | 你的訪談筆記、訪談逐字稿、內部備忘錄 |
| 網頁 | `.html` | 完整存檔的網頁（用瀏覽器「另存為」） |

### 3. （選用）寫一份 MANIFEST.md

如果檔名不夠清楚，可以在子目錄裡放一份 `MANIFEST.md` 標註每份檔案是什麼。沒寫也沒關係，agent 會用檔名 + PDF 第一頁推測，模糊時會主動問你一次。

範例：

```markdown
# 個人理財 SaaS A input manifest

- `2024-annual-report.pdf` — 2024 年報，公司 IR 網站下載
- `linkedin-employees-screenshot.png` — 2026-04-07 截圖，顯示員工數 47 人
- `interview-2026-03-notes.md` — 我與該公司 PM 的訪談筆記
```

## 規則

### Agent 怎麼處理這些檔案

1. **最高優先級**：drop zone 檔案的優先序高於任何 web 搜尋結果，但仍須遵守交叉驗證規則
2. **仍須交叉驗證**：drop zone 檔案算 1 個來源。涉及登記資料（董監事、資本額、地址）等關鍵欄位時，agent 仍會找至少 1 個獨立 web 來源比對
3. **引用格式**：報告中會以 `[來源: cases/{target}/input/{filename}]` 標註
4. **時間戳重要**：截圖類請在檔名或 MANIFEST 標註截圖日期，避免陳舊資料污染報告

### 隱私

`cases/{target}/` 子目錄被 `.gitignore` 排除，**不會推上 git**。你可以安心放年報、合約、訪談筆記這類敏感檔案。

只有這份 `README.md` 和 `.gitkeep` 會被 tracked。
