# 年報解析（Annual Report Analysis）

## 適用條件

| 規模分類 | 執行要求 | 備註 |
|---------|---------|------|
| 大型/上市 | **強制** | 未取得年報即進入 company-deep-dive，所有受影響維度將降級標註 |
| 中型 | 建議 | 有公開申報（如興櫃、公開發行）則執行；未公開發行則跳過 |
| 微型 | 跳過 | 不適用 |

## 階段目標

精準取得一份高密度文件（年報/10-K/事業報告書），提取 12 項關鍵數據點，產出結構化摘要直接餵入 company-deep-dive，並強制校正先前階段的搜尋誤差。

## 輸入

- 公司名稱、規模分類、上市市場、法律實體驗證結果
- 使用者是否已提供本地 PDF 路徑（來自預分析評估階段）

## 來源查找策略

### Step 1：查詢來源登記表

先讀取 `references/methodology/annual-report-sources.md`，根據公司所在市場取得對應的年報來源。

### Step 2：取得 PDF

按以下優先序嘗試，成功即停止：

1. **使用者已提供本地 PDF 路徑** → 直接 Read（最快、最可靠）
2. **公司 IR 網站 fetch** → `web_search "[公司名] investor relations annual report"` 或 `"[公司名] 投資人關係 年報"`，找到 PDF URL 後 `web_fetch`
3. **監管機構平台** → 依市場不同（台灣 MOPS、美國 EDGAR 等），`web_search` + `web_fetch`
4. **HTML 版年報** → 部分公司提供線上版年報，`web_fetch` HTML 頁面
5. **全部失敗** → 標註 `[年報未取得]`，列出已嘗試的來源和失敗原因，進入降級模式

### Step 3：解析

- PDF 檔案使用 Read 工具讀取（支援 PDF 頁碼指定）
- 年報通常 60-200 頁，**不要一次全讀**。先讀目錄頁（通常前 5 頁），定位各章節頁碼，再針對性讀取
- 優先讀取順序：致股東報告書（業績摘要）→ 營業概況（營收拆分）→ 財務狀況（損益表/資產負債表）→ 重大合約 → 董監事/股東 → 員工數據

## 必須提取的 12 項數據點

| # | 數據點 | 年報常見位置 | 重要性 |
|---|--------|------------|--------|
| 1 | **營收拆分**（依產品/事業群/地區） | 營業概況、業務內容 | 🔴 核心 — 搜尋結果常與年報有重大差異 |
| 2 | **毛利拆分**（依產品/事業群） | 財務績效分析 | 🔴 核心 |
| 3 | **營業外損益明細** | 財務績效分析 | 🔴 核心 — 識別一次性利益，避免 EPS 誤判 |
| 4 | **前五大客戶/供應商集中度** | 進銷貨客戶名單 | 🔴 核心 — 搜尋無法取得 |
| 5 | **完整股東名單** | 股本來源、主要股東 | 🔴 核心 — 注意「利用他人名義持有」欄位 |
| 6 | **董監持股變動** | 董事資料表 | 🟡 重要 |
| 7 | **關係人交易** | 財務報表附註 | 🟡 重要 |
| 8 | **員工數趨勢**（3 年、含部門拆分） | 從業員工 | 🟡 重要 — 判斷組織擴張/縮編方向 |
| 9 | **重大合約**（對象、到期日、自動續約狀態） | 重要契約 | 🔴 核心 — 「自動續約」vs「合約生效中」是關鍵差異 |
| 10 | **風險因素**（公司自行揭露） | 風險事項分析 | 🟡 重要 |
| 11 | **訴訟/或有負債** | 財務報表附註、風險事項 | 🟡 重要 |
| 12 | **研發支出**（金額+佔營收比） | 技術及研發概況 | 🟢 補充 |

## 搜尋預算

| search | fetch | 備註 |
|--------|-------|------|
| 5 | 3 | 目標是精準取得一份高密度文件，非廣泛搜尋。大部分預算用在定位 PDF URL |

## 產出格式

**不產出獨立報告。** 產出兩份結構化文件，作為 company-deep-dive 的直接輸入：

### 產出 1：年報數據摘要

```
## 年報數據摘要

**來源**：[年報名稱/URL]
**年度**：[財務年度]
**頁數**：[總頁數]
**解析日期**：[今天日期]

### 營收結構
[年報原始數據表格]

### 損益關鍵數字
[營收、毛利、營業費用、營業利益、業外損益、稅後淨利、EPS — 含前一年度比較]

### 供應商/客戶集中度
[前五大進貨/銷貨客戶代號、金額、佔比]

### 股東結構
[前十大股東、持股比例、特殊標記（如利用他人名義持有）]

### 員工數據
[總數、部門拆分、平均年齡/年資 — 含前一年度比較]

### 重大合約
[合約性質、對象、起訖日期、自動續約狀態、授權地區]

### 研發支出
[金額、佔營收比]

### 風險因素（公司自揭）
[摘要列表]
```

### 產出 2：與先前搜尋結果的衝突清單

```
## 衝突校正

| 維度 | 先前搜尋結果 | 年報數據 | 差異 | 採用 |
|------|------------|---------|------|------|
| [例] 營收結構 | 遊戲佔 90% | 遊戲佔 74%、勞務 18%、銷貨 8% | 顯著差異 | 年報 |
| ... | ... | ... | ... | ... |
```

此清單強制要求逐項比對。如無衝突，填寫「無衝突」。

## 降級模式

如果 PDF 取得全部失敗：

1. 標註 `[年報未取得]`，列出已嘗試的來源和失敗原因
2. 記錄在搜尋紀錄中，供品質 Review 階段檢視
3. 通知 Skill 層，由 Skill 決定是否提示使用者手動下載
4. company-deep-dive 將按「無年報數據」模式執行，受影響維度全部降級標註

## 完成標準

- [ ] 12 項數據點中至少 8 項已提取（未取得的標註原因）
- [ ] 衝突清單已完成（無衝突也要明確記錄）
- [ ] 年報來源和頁碼引用完整
- [ ] 數據摘要格式符合上述模板，可直接被 company-deep-dive 引用
