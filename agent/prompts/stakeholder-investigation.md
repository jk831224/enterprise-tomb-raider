# 利害關係人調查（Stakeholder Investigation）

## 階段目標

對目標公司的關鍵人物執行三層深度調查，產出組織權力結構圖、關係資本歸屬判斷、人事穩定性評估。

## 輸入

- 公司名稱、規模分類、法律實體驗證結果

## 三層調查框架

### 第一層：人物基本盤（所有規模）

- 創辦人/CEO/代表人：職業背景、前東家、產業經歷、公開言論
- 核心管理層（CTO、COO、CPO 等）：是否存在、背景
- 董監事：外部投資人？純合規？
- 關鍵技術/業務人物

### 第二層：關係網絡（中型以上必做，微型盡力）

- 師承與同門：同校、同公司、同創業社群 → 信任圈與派系
- 共事歷史：老戰友 vs 陌生組裝
- 投資人與管理層關係：董事席位、決策獨立性
- 離任者動向：和平分手 vs 路線衝突、是否做競品

### 第三層：個人行為影響鏈（關鍵人物）

- 副業與平行事業：利益衝突、資源分流
- 公開言論與戰略訊號：前後一致性
- 歷史決策追蹤：重大決策由誰主導、後果
- 訴訟與爭議
- 個人品牌經營：是否在為下一步鋪路

## 職稱校準規則

同一人物在不同來源出現不同職稱時：LinkedIn 自述 > 公司官網 > 新聞報導。多頭銜全部記錄，以最高層級為主（除非有明確降職證據）。所有「現任」標註「截至 [查詢日期]」。

## fetch 決策

1. 先 web_search「[公司] 新聞稿 OR blog OR 媒體中心」
2. **公司官方 blog / 新聞稿頁面 → 即使 snippet 看起來足夠，仍執行至少一次 fetch**（人事細節常藏在全文中，snippet 會遺漏）
3. 其他頁面：snippet 已帶出足夠資訊 → 不 fetch
4. snippet 截斷但有高密度內容 → 檢查 `fetch-policy.md` 黑名單 → 不在黑名單則 fetch
5. 在黑名單 → 搜更多關鍵詞取代

## 微型企業注意

- 10-50 人通常不設 C-Suite，搜不到 = 營運高度依賴創辦人（這本身是結論）
- 重點判斷：商業關係以組織建構 or 以個人為節點
- 注意關鍵人物是否同時經營獨立法人（顧問公司、工作室）

## 產出：三項判斷

1. **組織權力結構圖**：決策權分佈、集中度、管理層替換歷史與模式
2. **關係資本歸屬**：外部關係綁組織 or 個人、關聯實體分流風險
3. **人事穩定性**：留存率、離任模式（個別/批次、哪類先走）、替換週期、組織韌性
