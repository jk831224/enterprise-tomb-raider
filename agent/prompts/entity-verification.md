# 法律實體驗證（Entity Verification）

## 階段目標

在 30 秒內讓使用者知道這家公司是誰，同時完成法律實體的嚴格驗證。

## 搜尋策略

| 目標 | 搜尋樣式 |
|------|---------|
| 快速輪廓 | `[公司名] 是什麼` `[公司名] about` `[公司名] 官網` `[公司名] crunchbase` |
| 非主流快篩 | `[公司名] OR [負責人] site:ct.org.tw OR 校友 OR 受訪` |
| 法律實體（台灣） | `[公司名] 公司登記` `[公司名] 統一編號` + 商工登記 / twincn / twfile |
| 法律實體（海外） | `[公司名] incorporation` `[公司名] SEC filing` + 對應國家登記系統 |
| 關聯法人 | `[負責人] 公司` `[負責人] 負責人` `[負責人] 董事` + 同負責人相關資料 |
| 董監事變更歷史 | `"[公司名]" 董監事變更` `"[統編]" 董監事歷史` |
| 員工數三角定位 | `[公司] site:cake.me OR site:yourator.co OR site:104.com.tw` `"[公司]" site:linkedin.com employees` `[公司] 員工數 OR 人數 OR 團隊規模` |

## 員工數三角定位規則

至少從兩個獨立來源取得員工數。信任度排序：求職平台 > LinkedIn > 新聞報導 > TIEA/公會。若來源差異 >30%，標記 [待驗證] 並記錄各來源數字。

## 交叉比對規則

發現多個法律實體時，**至少滿足 3 項才可判定為同一事業體**：
1. 負責人相同
2. 共用品牌名/官網
3. 業務領域相關
4. 登記地址鄰近
5. 公開資料說明關係（官網、新聞稿提及子公司/關係企業）

## 完成標準

- [ ] 一句話定位 + 所屬產業 + 上市狀態 + 規模量級
- [ ] 法律實體：登記名稱 + 統編 + 資本額 + 設立日 + 負責人
- [ ] 董監事現任 + 變更歷史追蹤（追蹤異動頻率和模式）
- [ ] 關聯法人搜尋完成（含「未發現」也要記錄）
- [ ] 員工數至少兩個來源交叉比對
- [ ] 規模分類判定 + 判定依據
