# Decision Brief — 決策簡報產出

## 階段目標

基於已完成的研究報告和使用者 Profile，產出一份角色化的決策簡報。讓讀者知道「這對我意味著什麼」。

## 核心原則

1. **零搜尋預算**：不做任何 web_search 或 web_fetch。所有內容來自已產出的主報告。
2. **引用制**：每一個觀點必須指向主報告中的具體章節或數據。不引入報告外的資訊。
3. **證據等級繼承**：主報告中標記為 `[部分證據]` 或 `[推測，待驗證]` 的內容，在 Decision Brief 中必須保留原標記，不得升級。
4. **對話語氣**：用「你」稱呼讀者。直接、具體、不打官腔。
5. **判斷優先**：這不是摘要。讀者要的是「所以呢？」的答案。

## 角色鏡頭

根據 User Profile 的角色欄位，套用對應的分析鏡頭：

### 投資人（investor）
- **聚焦**：投資風險、估值訊號、退出路徑可行性、創辦人可信度
- **紅旗權重**：財務健康 > 市場天花板 > 管理層穩定度 > 法律風險
- **問題場景**：DD 會議、投委會報告
- **行動導向**：要不要投、投多少、什麼條件

### 求職者（job-seeker）
- **聚焦**：公司真實狀態 vs 對外形象、團隊穩定度與文化訊號、成長跑道、職涯天花板
- **紅旗權重**：人員流動率 > 燒錢速度 > 管理層更替頻率 > 產業衰退訊號
- **問題場景**：面試現場、offer 談判
- **行動導向**：要不要去、要問什麼、要談什麼條件

### 商業合作夥伴（partner）
- **聚焦**：履約能力、關係資本歸屬風險、對方的策略動機、談判籌碼
- **紅旗權重**：財務穩定度 > 客戶集中度 > 關鍵人物依賴 > 法律爭議
- **問題場景**：商務談判、合約條款設計
- **行動導向**：要不要合作、什麼條款保護自己、風險分攤機制

### 競品分析師（competitor-analyst）
- **聚焦**：核心優劣勢、策略走向預判、可攻擊缺口、護城河厚度
- **紅旗權重**：市佔趨勢 > 技術壁壘 > 人才流向 > 融資動態
- **問題場景**：策略會議、競品追蹤報告
- **行動導向**：對方下一步可能做什麼、我方該如何應對

### 其他（other）
- 根據 User Profile 的角色描述和決策情境，自行判斷最相關的分析角度
- 在簡報開頭說明你選擇的分析鏡頭及原因

## 產出結構

依照 `references/templates/decision-brief.md` 的六段結構產出：

1. **一句話判斷**
2. **對你最重要的三件事**
3. **紅旗警示**
4. **你的特殊關注回應**（僅當 profile 有填「特殊關注」）
5. **你該問的問題**（3-5 個場景化問題）
6. **下一步行動**（2-3 個具體動作）

## 完成標準

自我檢查以下項目，全部通過才算完成：

- [ ] 每一個觀點都引用了主報告的具體章節或數據
- [ ] 角色鏡頭與 User Profile 一致
- [ ] User Profile 中的所有「特殊關注」都已回應
- [ ] 問題具體到可以在真實場景中直接使用
- [ ] 下一步行動是具體動作，不是「建議進一步研究」
- [ ] 沒有引入主報告之外的資訊
- [ ] 證據等級標記完整繼承，未升級
- [ ] 語氣是對話式的「你」，不是學術報告口吻
