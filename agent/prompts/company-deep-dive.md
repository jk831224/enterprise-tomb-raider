# 公司深度分析（Company Deep Dive）

## 階段目標

將企業視為「商業機器」解構。整合前序成果（實體驗證、利害關係人、產業分析），產出完整公司報告。

## 輸入

- 公司名稱與規模分類
- 法律實體驗證結果、利害關係人調查結果、產業分析結果
- **年報數據摘要 + 衝突清單**（來自 annual-report-analysis，大型/上市公司適用）
- 分析目的與特殊關注

## 年報數據前置檢查（大型/上市公司適用）

進入本階段前，檢查是否已完成 annual-report-analysis：

**如果已完成**：使用年報數據摘要作為本階段的主要數據源。衝突清單中標記「採用年報」的維度，以年報數據為準，不再使用先前搜尋結果中的對應數字。

**如果未完成（大型/上市）**：以下維度將降級標註，並在報告開頭 metadata 中標註「⚠️ 本報告未取得年報原始數據，以下維度已降級標註。建議取得年報後補充更新。」

| 受影響維度 | 降級標註 | 原因 |
|-----------|---------|------|
| 營收拆分 | `[推測，待驗證]` | 搜尋結果可能過時或口徑不一致 |
| 供應商/客戶集中度 | `[資料缺失]` | 僅年報揭露前五大進銷貨客戶 |
| 關係人交易 | `[資料缺失]` | 僅年報附註揭露 |
| 重大合約細節（到期日、自動續約） | `[資料缺失]` | 年報「重要契約」章節為唯一來源 |
| 員工數趨勢 | `[部分證據]` | 僅有單一外部來源，缺部門拆分 |
| 股東實際控制比例 | `[部分證據]` | 可能遺漏「利用他人名義持有」 |

**如果不適用（微型/中型非公開）**：跳過此檢查，正常進入分析框架。

## 分析框架

### 1. 商業模式核心

**價值交換**：產品/服務、客戶痛點、收入模型（一次性/經常性、定價、多元性）
**成長引擎**：有機（產品創新、市場擴張、客戶滲透）+ 無機（併購、合資、策略聯盟）+ 可持續性
**競爭護城河**：網路效應、轉換成本、規模經濟、品牌、智財、監管壁壘、關係資本
- 回扣產業 KSF（如有）：護城河覆蓋了哪些 KSF？缺口在哪？

### 2. 利害關係人與組織運營

**利害關係人整合**（重新詮釋 Step 1.5 結果）：
- 權力結構如何影響決策品質和速度
- 關係資本歸屬對估值/合作價值的影響
- 人事穩定性對中長期發展的影響

**組織運營**（依 B2B/B2C/平台/SaaS 調整）：
- 前端（研發/產品/製造）：如何創造價值
- 市場端（行銷/銷售/客戶）：如何傳遞和捕獲價值
- 支援端（人資/財務/IT/法務）：如何維持運營
- 核心能力落在哪個環節

### 3. 財務健康度

**大型/上市**：三大報表（損益+資產負債+現金流）、資金結構
**中型**：融資輪次+主要投資人+燒錢率+跑道；或營收+成長+獲利
**微型**：現金流結構分析（循環模式、應收/應付錯配、大客戶依賴、黑字破產風險）

#### 微型企業：營收間接推估 [標記為推測，待驗證]

| 方法 | 做法 |
|------|------|
| A | 員工數 × 產業人均營收（從同業上市公司推估基準） |
| B | 資本額 + 融資歷史 → 年燒率 → 是否自給自足 |
| C | 客戶規模 × 產業平均客單價（如客戶名單可得） |
| D | 建議使用者查詢中華徵信所（付費，NTD 數百~數千/筆） |

取多種方法交集作為區間。

### 4. 盡職調查重點

- **財務面**：營收品質、客戶集中度、關係人交易
- **法務面**：進行中訴訟、合規風險、智財權
- **營運面**：PMF、客戶留存/流失、單位經濟效益（CAC、LTV）

## 產出格式

使用 `references/templates/company-report.md` 結構。明確標註：
- 哪些結論來自前序階段（交叉引用不重複）
- 哪些是本階段新增分析
- 規模分類如何影響分析策略選擇

## 完成標準

- [ ] 商業模式三要素（價值交換、成長引擎、護城河）已分析

- [ ] 財務分析與規模匹配（微型不套三大報表、大型不缺財報）
- [ ] 微型企業已執行營收間接推估（至少一種方法）
- [ ] DD 覆蓋財務+法務+營運三面向
