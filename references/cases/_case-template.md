# Case 標準結構模板

每次完成分析後，依此結構沉澱一份 case 檔案，供 agent 學習已知陷阱。

---

```markdown
# Case: [公司或產業名稱]

**分析日期**：YYYY-MM-DD
**分析路徑**：A / B
**規模分類**：微型 / 中型 / 大型
**分析目的**：[目的]

## 分析概要

[2-3 句話描述這次分析做了什麼、最終產出了什麼]

## 遇到的陷阱

### 陷阱 1: [陷阱名稱]

**現象**：[第一輪分析中遺漏或錯誤的具體內容]
**根因**：[為什麼會遺漏——搜尋策略不足、框架假設錯誤、工具使用不當等]
**修正方式**：[如何發現並修正]
**教訓**：[提煉為可複用的規則或檢查項]

[重複以上結構，覆蓋所有遇到的陷阱]

## 有效的搜尋策略

[記錄本次分析中特別有效的搜尋關鍵詞、資料來源、或搜尋路徑]

## 無效的搜尋策略

[記錄嘗試過但未產出有價值結果的搜尋方向，避免未來重複]

## 規模適配反思

[本次的規模分類是否正確？分析框架的調整是否合理？是否有過度分析或分析不足的章節？]

## 品質自評

[對照 quality-checklist.md，哪些項目通過、哪些項目未通過或有瑕疵]
```
