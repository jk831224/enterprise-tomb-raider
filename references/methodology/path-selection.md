# 路徑選擇邏輯（Path Selection）

## 兩條路徑

分析的組成模組相同（產業分析 + 公司分析），差別在進入順序和敘事方向。

### 路徑 A — 從產業看進去

**敘事方向**：從大到小。先建立產業地圖，再把公司放進地圖裡定位。

**觸發訊號**：
- 使用者提到的是產業名稱、產業趨勢、供應鏈分析等產業層級關鍵詞
- 沒有指定特定公司
- 例：「台灣的 SaaS 產業」「東南亞電商物流的供應鏈」「半導體產業現在怎麼樣」

**執行順序**：
1. 需求確認（Scoping）
2. 產業分析
3. 詢問是否深入特定公司
4. 公司分析（如使用者要求）

### 路徑 B — 從公司看出去

**敘事方向**：由點到面再回到點。先認識公司 → 展開產業全景 → 帶著產業脈絡重新理解公司的定位和價值。

**觸發訊號**：
- 使用者直接丟出一個公司名稱
- 例：「幫我看 Palantir」「分析 TSMC」「Stripe 的商業模式是什麼」

**執行順序**：
1. 快速定位（含法律實體驗證與關鍵利害關係人調查）
2. 以該公司為錨點的產業分析
3. 帶著產業脈絡的公司深度分析

## 判斷規則

| 使用者輸入特徵 | 路徑 | 理由 |
|--------------|------|------|
| 只提產業名稱，無特定公司 | A | 使用者想看全景 |
| 直接指定公司名稱 | B | 使用者已有目標 |
| 提到產業 + 「有哪些公司」 | A | 想透過產業發現公司 |
| 提到公司 + 「這個產業怎麼樣」 | B | 公司是起點，產業是背景 |
| 同時提到產業和公司 | B | 公司是分析焦點，產業是脈絡 |
| 模糊不清 | 直接問使用者 | 不猜測 |
