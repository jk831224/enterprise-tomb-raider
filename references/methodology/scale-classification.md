# 規模分類與分析策略

## 分類標準

| 分類 | 判定標準 | 典型特徵 |
|------|---------|---------|
| 微型 | 員工 <50、資本額 <5,000 萬 NTD、非上市 | 創辦人=管理層；決策集中；組織扁平 |
| 中型 | 員工 50-500、或 B 輪以上 | 部門分工；可能有外部董事 |
| 大型/上市 | 員工 >500、或已上市 | 完整組織；公開財報；機構投資人 |

## 策略差異

| 維度 | 微型 | 中型 | 大型 |
|------|------|------|------|
| **強制數據源** | 商業登記 + 求職平台 | 商業登記 + 求職平台 + 年報（如公開申報） | **商業登記 + 年報（強制）+ 財經平台** |
| 財務 | 現金流結構推論 | 融資輪次+估值 | 三大報表（以年報為準） |
| 組織 | 跳過，改做創辦人能力評估 | 標準框架 | 完整組織分析 |
| 利害關係人 | 關鍵人物+關係資本（核心） | 完整三層 | 完整三層+董事會 |
| 產業 | 簡化五力 | 標準五力 | 完整五力+數據 |
| DD | 法律實體+關鍵人物風險 | 標準框架 | 完整 DD |

> **年報強制規則**：大型/上市公司未取得年報即進入 company-deep-dive，視為分析品質不達標。系統會在 company-deep-dive 階段自動降級受影響的維度並標註（見 `agent/prompts/company-deep-dive.md` 的「年報數據前置檢查」）。中型企業若有公開申報年報，建議取得以提升品質，但非強制。

邊界情況：員工 <50 但資本額高 → 看業務模式（資本密集用中型策略）。規模不明 → 先用中型策略，過程中調整。
