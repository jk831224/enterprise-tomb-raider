# 規模分類與分析策略（Scale Classification）

## 分類標準

| 分類 | 判定標準 | 典型特徵 |
|------|---------|---------|
| 微型企業 | 員工 < 50 人、資本額 < 5,000 萬 NTD、非上市 | 創辦人 = 管理層 = 核心業務；決策高度集中；組織結構扁平或不存在 |
| 中型企業 | 員工 50-500 人、或已獲得 B 輪以上融資 | 有初步的部門分工；可能有外部董事；開始出現正式的管理流程 |
| 大型 / 上市企業 | 員工 > 500 人、或已上市 | 完整的組織架構；公開財報；機構投資人；正式的公司治理 |

## 分類對分析策略的影響

### 微型企業

**重點分析**：
- 關鍵人物調查（創辦人/負責人的背景、人脈、副業、雙重身份）
- 關係資本分析（客戶關係綁定在組織還是個人）
- 現金流結構性風險（先收後付 vs 先墊後收、大客戶依賴度）
- 法律實體結構（是否有關聯法人、資源分流風險）

**略過或簡化**：
- 三大報表分析（非上市微型企業財報不公開，硬套框架只產出猜測）
- 組織結構圖（因為創辦人 = 管理層 = 業務 = 一切）
- 管理層評估（改為創辦人個人能力與風險評估）

**特殊注意**：
- 如果搜尋不到 CTO/COO 等角色，這本身就是分析結論
- 注意「商業關係是以組織為單位建構的，還是以個人為節點建構的」

### 中型企業

**標準框架，適度調整**：
- 三大報表：如有公開數據則分析，如無則改用融資輪次和估值推估
- 組織分析：有意義，但可能不如大型企業完整
- 利害關係人調查：完整三層

### 大型 / 上市企業

**完整框架**：
- 三大報表為必要分析（公開數據可取得）
- 波特五力、KSF 分析要有數據支撐
- 利害關係人調查：完整三層，且可交叉比對公開的年報、法說會資訊
- 與同業的財務橫向比較

## 邊界情況

- 如果一家公司員工 < 50 但資本額 > 5,000 萬 → 看業務模式判斷。資本密集型（硬體、製造）可能是中型策略；輕資產型（顧問、SaaS）仍適用微型策略
- 如果一家公司已獲 A 輪融資但員工 < 30 → 以微型策略為主，但補充融資相關分析
- 如果規模資訊無法取得 → 先以中型策略執行，在過程中根據發現調整
