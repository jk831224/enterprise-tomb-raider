# 公司報告結構

Agent 依此大綱產出報告，根據規模分類調整各章節深度。

## Version History（首次分析即建立）

```markdown
## Version History
| 版本 | 日期 | 觸發 | 變更摘要 |
|------|------|------|---------|
| v1.0 | YYYY-MM-DD | `/company` 首次分析 | 基礎報告 |
```

版本命名規則：
- v1.0 = `/company` 或 `/recon` 首次產出
- v1.1, v1.2... = `/supplement` 增量更新
- v2.0 = 完整重新分析

## 必填章節

1. **摘要**（3-5 句，點出最關鍵的發現）
2. **公司基本資訊**（登記名稱、品牌名、一句話定位、產業、成立時間、上市狀態、規模、總部、法律實體表 + 交叉驗證結論）
3. **關鍵利害關係人**
   - 組織權力結構（決策權分佈、權力集中度、管理層替換歷史）
   - 關鍵人物檔案（背景、角色、關聯實體、公開言論、爭議）
   - 關係資本歸屬（外部關係綁組織 or 個人、分流風險）
   - 人事穩定性（留存率、離任模式、替換週期、組織韌性）
   - 人事與公司事件交叉時間軸
4. **商業模式**（價值交換機制 + 成長引擎 + 競爭護城河，回扣產業 KSF）
5. **組織運營**（前端/市場端/支援端，核心能力定位）
6. **財務健康度**
   - 微型 → 現金流結構分析（循環模式、應收/應付錯配、大客戶依賴、黑字破產風險）
   - 中型 → 融資與估值（輪次表、燒錢率、資金跑道）
   - 大型 → 三大報表（損益、資產負債、現金流、資金結構）
7. **盡職調查重點**（財務面 + 法務面 + 營運面）
8. **綜合評估**（機會 + 風險 + 針對分析目的的結論）
9. **OSINT 天花板**（3-5 個「只有內部人才能回答、但答案會大幅改變分析結論」的問題）
10. **附錄**（搜尋紀錄摘要 + 證據等級依 `output-quality.md`）

## 每章必備

- 來源標註（URL 或報告名+年份）
- 證據等級標記（依 `.claude/rules/output-quality.md`）
- 規模分類對分析深度的影響說明
- 人物職稱標註「截至 [查詢日期]」

## Metadata

報告開頭必含：分析日期、規模分類、分析目的、分析路徑。
