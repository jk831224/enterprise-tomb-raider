# 決策簡報模板

> Decision Brief — 基於完成的研究報告 + 使用者 Profile，產出角色化的行動解讀。

## Metadata

- **對應報告**：[主報告檔名]
- **使用者角色**：[investor / job-seeker / partner / competitor-analyst / other]
- **簡報日期**：YYYY-MM-DD

---

## 1. 一句話判斷

> 以 [角色] 的視角，[研究目標] [核心判斷]。

格式：直接、明確、不含糊。不是摘要，是判斷。

---

## 2. 對你最重要的三件事

每一項必須：
- 引用主報告的具體章節或數據
- 說明為什麼「這個角色」該在意這件事
- 用 2-3 句話講完

```
### [發現標題]
**報告來源**：[章節名稱 / 具體數據點]
**為什麼你該在意**：[角色視角的解讀]
```

---

## 3. 紅旗警示

按該角色的風險偏好排序。每個紅旗：
- **事實**：報告中的具體發現
- **推論**：這個事實意味著什麼
- **嚴重程度**：🔴 高 / 🟡 中 / 🟢 低
- **證據等級**：繼承主報告的標記（充分證據 / 部分證據 / 推測待驗證）

沒有紅旗也要明確寫「未發現顯著紅旗」，並說明為什麼。

---

## 4. 你的特殊關注回應

> 僅當 User Profile 的「特殊關注」欄位有內容時產出此段。

逐項回應使用者 profile 中列出的每個關注點：
- **關注點**：[使用者原文]
- **報告中的對應發現**：[引用]
- **判斷**：安全 / 需留意 / 警示

---

## 5. 你該問的問題

3-5 個問題。必須：
- 場景化（投資人：DD 會議上問；求職者：面試時問；合作方：商務談判時問）
- 針對報告中的資訊缺口或待驗證項目
- 具體到可以直接使用，不是「你覺得公司前景如何」這種空泛問題

---

## 6. 下一步行動

2-3 個具體動作。不是「建議進一步研究」，而是：
- 具體去做什麼（聯繫誰、查什麼、準備什麼）
- 優先順序
- 時效性（如果有的話）
